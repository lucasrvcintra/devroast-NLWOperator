# PADRÕES DE COMPONENTES UI

ESTE DOCUMENTO DEFINE OS PADRÕES PARA CRIAÇÃO DE COMPONENTES DE UI NO PROJETO.

## ESTRUTURA DE ARQUIVOS

```
SRC/
└── COMPONENTS/
    └── UI/
        ├── BUTTON.TSX
        ├── INPUT.TSX
        └── ...
```

## REGRAS PARA CRIAÇÃO DE COMPONENTES

### 1. NAMED EXPORTS

SEMPRE USE NAMED EXPORTS. NUNCA USE DEFAULT EXPORT.

```TXS
// ✅ CORRETO
EXPORT INTERFACE BUTTONPROPS { ... }
EXPORT CONST BUTTON = FORWARDREF<...>(...)

// ❌ ERRADO
EXPORT DEFAULT FUNCTION BUTTON() { ... }
```

### 2. EXTENSÃO DE PROPS NATIVAS

ESTENDA AS PROPRIEDADES NATIVAS DO ELEMENTO HTML CORRESPONDENTE.

```TXS
IMPORT { TYPE BUTTONHTMLATTRIBUTES, FORWARDREF } FROM "REACT";

EXPORT INTERFACE BUTTONPROPS EXTENDS BUTTONHTMLATTRIBUTES<HTMLBUTTONELEMENT> {}
```

### 3. TAILWIND VARIANTS

USE `TAILWIND-VARIANTS` PARA CRIAR VARIANTES DE COMPONENTES.

```TXS
IMPORT { TV, TYPE VARIANTPROPS } FROM "TAILWIND-VARIANTS";

CONST BUTTONVARIANTS = TV({
  BASE: "CLASSES-BASE-COMUNS",
  VARIANTS: {
    VARIANT: {
      DEFAULT: "CLASSES-DEFAULT",
      SECONDARY: "CLASSES-SECONDARY",
    },
    SIZE: {
      SM: "H-9 PX-4",
      LG: "H-11 PX-8",
    },
  },
  DEFAULTVARIANTS: {
    VARIANT: "DEFAULT",
    SIZE: "DEFAULT",
  },
});
```

### 4. MERGE DE CLASSES

O `TAILWIND-VARIANTS` JÁ FAZ MERGE DAS CLASSES AUTOMATICAMENTE. PASSE `CLASSNAME` DIRETAMENTE NA FUNÇÃO DE VARIANTES.

```TXS
// ✅ CORRETO - TAILWIND-VARIANTS FAZ O MERGE
<BUTTON CLASSNAME={BUTTONVARIANTS({ VARIANT, SIZE, CLASSNAME })} />

// ❌ ERRADO - NÃO USE TAILWIND-MERGE COM VARIANTS
IMPORT { CN } FROM "@/LIB/UTILS";
<BUTTON CLASSNAME={CN(BUTTONVARIANTS({ VARIANT, SIZE }), CLASSNAME)} />
```

### 5. CLASSES CSS CANÔNICAS

USE A SINTaxe CANÔNICA DO TAILWIND V4 PARA VARIÁVEIS CSS:

```TXS
// ✅ CORRETO - SINTaxe CANÔNICA
"BG-(--ACCENT-GREEN)"
"TEXT-FOREGROUND"  // PARA CORES SIMPLES

// ❌ ERRADO - SINTaxe ANTIGA
"BG-[VAR(--ACCENT-GREEN)]"
"TEXT-[VAR(--FOREGROUND)]"
```

### 6. FORWARD REF

USE `FORWARDREF` PARA PERMITIR REF DO ELEMENTO NATIVO.

```TXS
CONST COMPONENT = FORWARDREF<HTMLBUTTONELEMENT, COMPONENTPROPS>(
  ({ CLASSNAME, VARIANT, ...PROPS }, REF) => {
    RETURN (
      <BUTTON
        CLASSNAME={COMPONENTVARIANTS({ VARIANT, CLASSNAME })}
        REF={REF}
        {...PROPS}
      />
    );
  }
);

COMPONENT.DISPLAYNAME = "COMPONENT";
```

### 7. TEMPLATE DE COMPONENTE

```TXS
IMPORT { TYPE ELEMENTHTMLATTRIBUTES, FORWARDREF } FROM "REACT";
IMPORT { TV, TYPE VARIANTPROPS } FROM "TAILWIND-VARIANTS";

CONST COMPONENTVARIANTS = TV({
  BASE: "CLASSES-BASE-OBRIGATÓRIAS",
  VARIANTS: {
    VARIANT: {
      DEFAULT: "CLASSES-VARIANT-DEFAULT",
      SECONDARY: "CLASSES-VARIANT-SECONDARY",
    },
    SIZE: {
      SM: "H-9 PX-4",
      LG: "H-11 PX-8",
    },
  },
  DEFAULTVARIANTS: {
    VARIANT: "DEFAULT",
    SIZE: "DEFAULT",
  },
});

EXPORT INTERFACE COMPONENTPROPS
  EXTENDS ELEMENTHTMLATTRIBUTES<HTMLDIVELEMENT>,
    VARIANTPROPS<typeof componentVariants> {}

CONST COMPONENT = FORWARDREF<HTMLDIVELEMENT, COMPONENTPROPS>(
  ({ CLASSNAME, VARIANT, SIZE, ...PROPS }, REF) => {
    RETURN (
      <DIV
        CLASSNAME={COMPONENTVARIANTS({ VARIANT, SIZE, CLASSNAME })}
        REF={REF}
        {...PROPS}
      />
    );
  },
);

COMPONENT.DISPLAYNAME = "COMPONENT";

EXPORT { COMPONENT, COMPONENTVARIANTS };
```

## FERRAMENTAS

- **TAILWIND-VARIANTS**: PARA CRIAR VARIANTES DE COMPONENTES
- **CLSX**: PARA CONDIÇÕES BOOLEANAS (SE NECESSÁRIO)
- **TAILWIND-MERGE**: APENAS PARA MERGE DE CLASSES MANUAL (NÃO USE COM VARIANTS)

## DESIGN SYSTEM

USE AS VARIÁVEIS CSS DEFINIDAS NO ARQUIVO `.PEN`:

- CORES: `--ACCENT-GREEN`, `--PRIMARY`, `--DESTRUCTIVE`, ETC.
- BORDERS: `--BORDER`, `--RING`
- BACKGROUNDS: `--BACKGROUND`, `--FOREGROUND`
- RADIUS: `--RADIUS-M`, `--RADIUS-PILL`

EXEMPLO:
```TXS
CONST BUTTONVARIANTS = TV({
  BASE: "BG-(--PRIMARY) TEXT-WHITE ROUNDED-[VAR(--RADIUS-M)]",
});
```

## FONTES

- **MONOSPACE**: USE `FONT-MONO` COM JETBRAINS MONO (CONFIGURADO NO TAILWIND)
- **SANS**: USE `FONT-SANS` COM FONTE PADRÃO DO SISTEMA (NÃO PREISA CONFIGURAR)

PARA USAR MONOSPACE:
```TXS
<BUTTON CLASSNAME="FONT-MONO">ROAST MY CODE</BUTTON>
```
