# Specification Format

Criar spec antes de implementar. Arquivo: `@specs/<feature-name>.md`

## Estrutura

```
# <Feature> — Specification

> Breve descricao do que a feature faz

## Contexto
Explicacao do problema que resolve e como se encaixa no produto

## Decisoes de design
Principais escolhas tecnicas e o porquê

## Estrutura de arquivos
Arquivos que serao criados/modificados

## To-do de implementacao
- [ ] Tarefa 1
- [x] Tarefa concluida
```

## Regras

- Usar titulo em ingles com "— Specification"
- Incluir codigo executavel (schemas, queries, configs)
- Diagramas quando necessario (ER, fluxo)
- Manter ate 2 paginas
